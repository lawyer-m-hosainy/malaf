🔴 المرحلة 1 — إصلاح التكرار وتوحيد مسار الفواتير
السياق
يوجد حالياً فاتورتان بنفس الغرض:

Modal بسيط في صفحة "المالية والفواتير": (اسم عميل + مبلغ فقط) — زر "حفظ وحساب الضريبة آلياً"
Modal مفصل في صفحة "الفاتورة الإلكترونية ETA": (رقم ضريبي + كود نشاط + ضريبة جدول 10%)

المطلوب
1. احذف الـ modal البسيط "إصدار فاتورة ضريبية جديدة" من صفحة FinancialPage بالكامل.
2. في صفحة FinancialPage، غيّر زر "إنشاء فاتورة ضريبية" ليعمل كـ Link يوجه المستخدم مباشرة إلى /invoices/eta:

// بعد

</Button>
3. في صفحة FinancialPage، الـ cards العلوية تبقى كما هي لكن أضف Quick Stats تجلب من Supabase:

4. البيانات تُجلب من Supabase بـ single query محسّنة:
sql-- Dashboard summary query
WITH monthly_revenue AS (
  SELECT DATE_TRUNC('month', created_at) as month,
         SUM(total_amount) as revenue
  FROM invoices
  WHERE tenant_id = auth.uid()
    AND status = 'paid'
    AND created_at >= NOW() - INTERVAL '6 months'
  GROUP BY 1
),
monthly_expenses AS (
  SELECT DATE_TRUNC('month', created_at) as month,
         SUM(amount) as expenses
  FROM expenses
  WHERE tenant_id = auth.uid()
    AND created_at >= NOW() - INTERVAL '6 months'
  GROUP BY 1
)
SELECT r.month, r.revenue, COALESCE(e.expenses, 0) as expenses
FROM monthly_revenue r
LEFT JOIN monthly_expenses e USING (month)
ORDER BY 1;

RLS Policies (نمط موحد لجميع الجداول الجديدة)
sql-- تطبيق على: trust_transactions, payment_plans, payment_installments
ALTER TABLE {table_name} ENABLE ROW LEVEL SECURITY;

CREATE POLICY "select_own" ON {table_name} FOR SELECT
  USING (tenant_id = auth.uid());

CREATE POLICY "insert_own" ON {table_name} FOR INSERT
  WITH CHECK (tenant_id = auth.uid());

CREATE POLICY "update_own" ON {table_name} FOR UPDATE
  USING (tenant_id = auth.uid());
