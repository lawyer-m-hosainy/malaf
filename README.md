# ⚖️ مَلَف (Malaf) - المذكرة الشارحة للمنصة القانونية المتكاملة
[![CI/CD](https://github.com/lawyer-m-hosainy/malaf/actions/workflows/ci-cd.yml/badge.svg)](https://github.com/lawyer-m-hosainy/malaf/actions/workflows/ci-cd.yml)

**مَلَف (Malaf)** هو نظام إيكولوجي رقمي متكامل (SaaS) مصمم لرقمنة العمل القانوني في جمهورية مصر العربية. يتجاوز كونه مجرد برنامج لإدارة القضايا، ليصبح "نظام تشغيل قانوني" يدمج بين الأتمتة الإجرائية، الذكاء الاصطناعي، والامتثال الضريبي والقانوني الكامل.

---

## 🏗️ التصميم الهندسي للنظام (Architecture)

يعتمد النظام على بنية تحتية سحابية حديثة تضمن التوسع والأمان الفائق:

### 1. البنية الأمامية (Frontend)
*   **الإطار البرمجي:** React 19 مع استخدام Vite كأداة بناء سريعة.
*   **إدارة الحالة:** مزيج بين `Zustand` للملحظات اللحظية و `React Query` للتعامل مع البيانات القادمة من السيرفر مع ميزة التخزين المؤقت (Caching).
*   **تصميم الواجهة:** نظام `Tailwind CSS 4` مع مكتبة `shadcn/ui` لتوفير واجهة مستخدم (UI/UX) عصرية تدعم اللغة العربية (RTL) بشكل أصيل.
*   **الأداء:** تحقيق نتائج ممتازة في `Core Web Vitals` (LCP < 2.5s, TTFB < 0.2s) عبر تقنيات Code Splitting و Lazy Loading.

### 2. البنية الخلفية وقاعدة البيانات (Backend & DB)
*   **قاعدة البيانات:** PostgreSQL مستضافة عبر `Supabase`.
*   **تعدد المستأجرين (Multi-Tenancy):** تطبيق مبدأ العزل التام للبيانات باستخدام **Row Level Security (RLS)**، حيث يتم ربط كل سجل بـ `organization_id` ولا يمكن لأي مستخدم الوصول لبيانات خارج نطاق مكتبه على مستوى نواة قاعدة البيانات.
*   **الحوسبة السحابية:** استخدام `Edge Functions` (Deno runtime) لتنفيذ المهام الحساسة (مثل التشفير وإرسال الإشعارات) بعيداً عن متصفح العميل.

## 🧩 الأقسام والوحدات الوظيفية (Platform Modules)

تم تقسيم المنصة إلى وحدات تخصصية تغطي كافة جوانب العمل القانوني:

### � الوحدة الأساسية (Core Management)
*   **لوحة التحكم (Dashboard):** عرض إحصائي شامل للقضايا، الجلسات القادمة، والتدفقات المالية.
*   **إدارة القضايا (Cases):** أرشفة ذكية للقضايا (مدني، جنائي، إداري) مع تتبع رقمي كامل.
*   **إدارة الموكلين (Clients):** قاعدة بيانات شاملة للموكلين تشمل التوكيلات (POA) وسجل التواصل.
*   **الأجندة الرقمية (Calendar):** مزامنة جلسات المحاكم والمواعيد النهائية القانونية.

### ⚖️ الوحدات المتخصصة (Specialized Tracks)
*   **القضاء الإداري (State Council):** تتبع دعاوى مجلس الدولة ومواعيد الطعون الإدارية.
*   **المحاكم الاقتصادية (Economic Court):** إدارة النزاعات التجارية والمالية.
*   **قضايا الأسرة (Family Courts):** قسم خاص لدعاوى الأحوال الشخصية.
*   **الملكية الفكرية (IP Management):** تسجيل العلامات التجارية وبراءات الاختراع.
*   **الشهر العقاري (Real Estate):** متابعة إجراءات التسجيل والتوثيق.

### � وحدة الإدارة المالية والضريبية (Legal Finance)
*   **الفاتورة الإلكترونية (ETA):** تكامل مباشر مع منظومة الضرائب المصرية لإصدار الفواتير المعتمدة.
*   **حسابات الأمانات (Trust Accounts):** نظام محاسبي لفصل أموال الموكلين عن أتعاب المكتب.
*   **خطة الدفع (Payment Plans):** جدولة الأتعاب وتحصيلها عبر بوابات الدفع الرقمية.

### 🤖 وحدة الذكاء الاصطناعي (AI & Knowledge)
*   **محلل الوثائق (AI Document Analyzer):** قراءة ملفات PDF واستخراج النقاط القانونية الجوهرية.
*   **المكتبة القانونية (Law Library):** قاعدة بيانات محدثة للتشريعات والقوانين المصرية.
*   **فحص التعارض (Conflict Check):** خوارزمية ذكية للتأكد من عدم وجود تعارض مصالح قبل قبول القضية.

## ⚙️ الخوارزميات والمنطق البرمجي (Core Algorithms)

### 1. خوارزمية التشفير السيادي (AES-256-GCM)
لحماية البيانات الحساسة (مثل مبالغ القضايا وهويات الموكلين):
*   يتم تشفير البيانات عبر `Edge Functions` لضمان عدم تعرض مفتاح التشفير (Master Key) للجهة الأمامية.
*   استخدام **Client-side LRU Cache** لفك التشفير في الذاكرة المؤقتة لتحسين تجربة المستخدم وتقليل استهلاك موارد السيرفر.

### 2. نظام قاطع الدائرة (Circuit Breaker)
لضمان استقرار المنصة عند التعامل مع خدمات خارجية (مثل بوابة الدفع أو بوابة الضرائب):
*   في حال فشل الخدمة الخارجية (مثلاً 5 مرات متتالية)، يقوم النظام بفتح الدائرة (OPEN) وتحويل المستخدم للوضع البديل (Fallback) لمنع انهيار النظام.
*   إعادة المحاولة التلقائية (Half-Open) بعد فترة تهدئة (Cooldown) للتأكد من عودة الخدمة.

### 3. خوارزمية المحاسبة القانونية
*   حساب تلقائي لضريبة القيمة المضافة (14% طبقاً للقانون المصري).
*   نظام **Exponential Backoff** لإعادة محاولة إرسال الفواتير لمنظومة الضرائب في حالة فشل الاتصال المؤقت.

## �️ الأمن والامتثال (Security & Compliance)

*   **القانون 151 لسنة 2020:** المنصة متوافقة تماماً مع قانون حماية البيانات الشخصية المصري، مع وجود سجل معالجة بيانات (Data Processing Register) آلي.
*   **سجل التدقيق (Audit Logs):** توثيق كل عملية (إضافة، تعديل، حذف) مع تسجيل هوية المستخدم، الوقت، والجهاز المستخدم.
*   **الحماية من الثغرات:** تطبيق معايير **OWASP Top 10** مع حماية خاصة ضد ثغرات الوصول (Broken Access Control) عبر سياسات RLS الصارمة.

نلتزم بأعلى معايير الهندسة البرمجية لضمان استدامة النظام:
*   **التعقيد البرمجي (Cyclomatic Complexity):** الحفاظ على درجة تعقيد أقل من 10 لكل دالة لضمان سهولة الاختبار والصيانة.
*   **التوثيق الذاتي (JSDoc):** توثيق كامل لكل الوظائف البرمجية مع شرح المنطق القانوني والتقني خلفها.

## 🚀 التشغيل والتطوير (Getting Started)

### المتطلبات الأساسية
*   Node.js +20
*   Supabase CLI
*   حساب في منظومة الضرائب المصرية (لبيئة الاختبار)

### خطوات التشغيل
1.  تنصيب الاعتمادات: `npm install`
2.  إعداد المتغيرات البيئية: `cp .env.example .env.local`
3.  تشغيل بيئة التطوير: `npm run dev`
4.  تشغيل الاختبارات: `npm test`

## 🧪 دليل ومنهجية الاختبار (Testing Playbook)

يعتمد مشروع **مَلَف** على منهجية "الاختبار متعدد الطبقات" لضمان استقرار المنصة القانونية. إليك الدليل التفصيلي لتشغيل ومتابعة الاختبارات:

### 1. المتطلبات التقنية والأدوات
*   **Node.js:** إصدار 20 أو أحدث.
*   **Vitest:** للاختبارات الأحادية واختبارات التكامل (Fast Unit Testing).
*   **Playwright:** للاختبارات الوظيفية الشاملة (E2E Testing).
*   **Stryker.js:** لاختبار الطفرات (Mutation Testing) وقياس جودة الاختبارات نفسها.
*   **Sentry:** لمراقبة الأخطاء في بيئة الإنتاج.

### 2. أنواع الاختبارات وخطوات التنفيذ

#### أ. الاختبارات الأحادية واختبارات التكامل (Unit & Integration)
تستهدف التأكد من صحة المنطق البرمجي والخوارزميات (مثل حساب الضريبة وتشفير البيانات).
*   **الأمر:** `npm run test`
*   **الأمر (مع التغطية):** `npm run test:coverage`
*   **التحقق:** يجب أن تجتاز جميع الاختبارات بنسبة 100%، مع تغطية كود (Coverage) لا تقل عن 80%.

#### ب. الاختبارات الوظيفية الشاملة (End-to-End)
تستهدف محاكاة رحلة المستخدم الحقيقية (تسجيل الدخول، إضافة قضية، إصدار فاتورة).
*   **المتطلبات:** تشغيل السيرفر محلياً (`npm run dev`).
*   **الأمر:** `npm run test:e2e`
*   **التحقق:** مراجعة تقارير HTML المولدة في مجلد `playwright-report`.

#### ج. اختبار الطفرات (Mutation Testing)
يستهدف التأكد من أن الاختبارات قوية بما يكفي لاكتشاف أي تغييرات خاطئة في الكود.
*   **الأمر:** `npm run test:mutation`
*   **التحقق:** يجب أن يكون "Mutation Score" أعلى من 70%.

#### د. اختبارات الأمان والذكاء الاصطناعي (AI & Security)
تستهدف التأكد من حماية المطالبات (Prompts) وجودة مخرجات الـ AI.
*   **الأمر:** `npx vitest run src/services/ai/prompt-sanitizer.test.ts`
*   **التحقق:** التأكد من حجب جميع محاولات الحقن (Injection) بنجاح.

### 3. التعامل مع الأخطاء وتوثيق النتائج
*   **عند ظهور خطأ:** 
    1. راجع سجلات التيرمينال (Logs) لتحديد مكان الخطأ.
    2. استخدم `console.log` أو أدوات Debugging في المتصفح.
    3. تأكد من أن المتغيرات البيئية (`.env.local`) محدثة وصحيحة.
*   **التوثيق:** يتم تسجيل نتائج الاختبارات آلياً في سجل التدقيق (Audit Logs) وعند استخدام GitHub Actions في كل PR.
