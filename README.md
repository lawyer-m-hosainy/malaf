# ⚖️ مَلَف (Malaf) - منصة الإدارة القانونية الذكية

**مَلَف (Malaf)** هي منصة برمجية خدمية (SaaS) مصممة خصيصاً لمكاتب المحاماة في جمهورية مصر العربية. المنصة تدمج بين الذكاء الاصطناعي وإدارة القضايا لتقديم حل تقني شامل يربط المحامي بموكليه، وينظم الشؤون المالية، ويحمي البيانات بأعلى المعايير الأمنية.

---

## 🚀 الرؤية والهدف (The Vision)
تهدف المنصة إلى رقمنة قطاع المحاماة المصري بالكامل، والانتقال من الإدارة الورقية العشوائية إلى لوحة تحكم رقمية واحدة. حيث تقوم المنصة بأتمتة المهام الروتينية، صياغة العقود بالذكاء الاصطناعي، ومتابعة قضايا التنفيذ المالي، ليتفرغ المحامي للترافع الاستراتيجي.

---

## ✨ الميزات الحصرية التنافسية (Key Features)

### 1. 🤖 المحامي الذكي (AI Legal Assistant)
نظام ذكاء اصطناعي مدمج يعمل بـ `Gemini API` ويحتوي على مسار احتياطي للعمل دون اتصال بالخادم (Fallback Mock System):
- **المستشار القانوني الآلي:** يقدم استشارات مبدئية استناداً إلى الأنظمة المصرية (قانون العقود، قانون العمل، إلخ).
- **صانع العقود الذكي:** توليد مسودات عقود قانونية معتمدة بضغطة زر.
- **محلل الوثائق:** تحليل العقود والمستندات لاستخراج نقاط القوة والضعف والمخاطر.

### 2. 🛡️ بنية أمنية مصرفية (Bank-Grade Security)
- **تشفير AES-256:** جميع البيانات الحساسة (الهويات، المبالغ المالية، ملفات القضايا) يتم تشفيرها وفك تشفيرها محلياً في المتصفح قبل إرسالها إلى قواعد البيانات.
- **العزل الشامل للمستأجرين (Multi-Tenancy Isolation):** كل مكتب محاماة (Tenant) لديه بيئة بيانات معزولة تماماً في `Supabase` بفضل سياسات أمان مستوى الصف (RLS Policies)، مما يضمن عزل البيانات بشكل كامل بين المكاتب.
- **سجلات الأمان والتدقيق (Audit Logs):** تتبع دقيق لأي إجراء مالي أو عرض لمعلومات حساسة (مثل ملفات التنفيذ) مع تسجيل (وقت الدخول، واسم الموظف).

### 3. 💼 إدارة مالية وضريبية (Financial & Tax Management)
- **إدارة حسابات الأمانات:** نظام شفاف لإدارة مبالغ العملاء وفصلها عن حسابات المكتب.
- **الفاتورة الإلكترونية المصرية (ETA):** نظام مدمج لإصدار فواتير ضريبية متوافقة مع متطلبات مصلحة الضرائب المصرية بضريبة 14%.

### 4. ⚡ بنية تحتية سحابية متقدمة (Modern Infrastructure)
- **البحث السريع:** دعم البحث المتقدم والفلترة الفورية لآلاف السجلات بفضل الفهرسة الذكية.
- **تكامل مع بوابات التقاضي:** جاهزية للربط مع بوابة التقاضي الإلكتروني، مجلس الدولة، والمحاكم الاقتصادية.

## 🛠️ التقنيات المستخدمة (Tech Stack)

| التقنية (Technology) | الغرض (Purpose) |
| :--- | :--- |
| **React 19 + Vite** | واجهة مستخدم سريعة جداً وتفاعلية (Frontend) |
| **Tailwind CSS 4 + shadcn/ui** | تصميم عصري، سلس، ومريح للعين مع دعم الوضع الليلي (Dark Mode) |
| **Zustand** | إدارة حالة التطبيق محلياً لضمان السرعة الفائقة وتقليل التحميل على السيرفر |
| **Supabase** | قاعدة البيانات الأساسية (PostgreSQL) مع دعم RLS (Row Level Security) |
| **Firebase Auth** | نظام المصادقة وإدارة المستخدمين (Authentication) |
| **Crypto-JS (AES-256)** | تشفير البيانات الحساسة من طرف إلى طرف (End-to-End Encryption) |
| **Zod** | تحقق دقيق من صحة مدخلات المستخدم قبل الإرسال لمنع الاختراقات |

## 💻 طريقة التشغيل المحلية (Local Development)

بالنسبة للمطورين أو المستثمرين الراغبين في معاينة المشروع محلياً، يمكن اتباع الخطوات التالية:

**1. تنصيب الحزم:**
\`\`\`bash
npm install
\`\`\`

**2. إعداد متغيرات البيئة:**
قم بإنشاء ملف `.env.local` في الجذر، وأضف المفاتيح التالية (احصل عليها من Firebase / لوحة التحكم):
\`\`\`env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_ENCRYPTION_KEY=your_secret_encryption_key
\`\`\`

**3. تشغيل الواجهة الأمامية (Frontend):**
\`\`\`bash
npm run dev
\`\`\`

**4. لتشغيل الخادم الخلفي للذكاء الاصطناعي (اختياري):**
\`\`\`bash
# في نافذة طرفية منفصلة
node server.js
\`\`\`
*(ملاحظة: النظام يحتوي على ميزة "البديل الذكي المحلي" الذي يضمن عمل ميزات الذكاء الاصطناعي وتوليد العقود بأمثلة واقعية حتى في حال عدم تشغيل الخادم).*
