# إجراءات استعادة الفواتير المرفوضة من ETA

### 1. الكشف عن الرفض
- **الإشعارات:** يرسل النظام تنبيهاً فورياً (WhatsApp/In-app) للمحاسب أو المحامي المسؤول عند تغيير حالة الفاتورة إلى "مرفوضة".
- **لوحة التحكم:** تظهر شارة حمراء (Badge) على تبويب "الفاتورة الإلكترونية" مع عدد الفواتير المرفوضة.

### 2. تحليل سبب الرفض
عند فتح الفاتورة المرفوضة في واجهة [ETAInvoicing.tsx](file:///e:/malaf/src/views/ETAInvoicing.tsx)، يظهر جدول الأخطاء المُعاد من المنظومة مترجماً:
- `Invalid Tax ID` -> "رقم التسجيل الضريبي للموكل غير صحيح."
- `Invalid Item Code` -> "كود الخدمة المستخدم غير مسجل في المنظومة."
- `Total Mismatch` -> "خطأ في حسابات الضريبة أو الإجمالي."

### 3. التعديل وإعادة الإرسال
1. النقر على زر **"تعديل وإعادة إرسال"** في تفاصيل الفاتورة.
2. تفتح نافذة [InvoiceForm.tsx](file:///e:/malaf/src/views/finance-components/InvoicePreview.tsx) مع تحديد الحقول المسببة للخطأ.
3. بعد التصحيح، يتم النقر على **"إرسال"**؛ يقوم النظام بتوليد `internalID` جديد (نسخة جديدة) مع الاحتفاظ بربطها بالفاتورة القديمة للأرشفة.

### 4. التصعيد (Escalation)
إذا رُفضت الفاتورة للمرة الثانية لنفس السبب:
- يتم حظر إعادة الإرسال التلقائي.
- تظهر رسالة "يرجى التواصل مع الدعم الفني للمنصة أو مراجعة محاسب المكتب القانوني".
- يتم إرسال تقرير الخطأ تلقائياً لفريق الدعم في "مَلَف".
