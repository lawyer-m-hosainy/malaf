# دليل نشر منصة "ملف" (Deployment Guide)

أنت الآن جاهز لنشر المنصة لتصبح متاحة عبر الإنترنت. اتبع هذه الخطوات بتركيز:

## أولاً: النشر على Render.com (السيرفر والواجهة)

1. قم بإنشاء حساب على [Render.com](https://render.com) واربطه بحساب GitHub الخاص بك.
2. اضغط على **New** ثم اختر **Web Service**.
3. اختر مستودع (Repository) الخاص بمشروع `malaf`.
4. أدخل الإعدادات التالية:
   - **Name:** `malaf-platform` (أو أي اسم تفضله).
   - **Runtime:** `Node`.
   - **Build Command:** `npm install && npm run build`.
   - **Start Command:** `node server.js`.

## ثانياً: إضافة المتغيرات البيئية (Environment Variables)

هذه أهم خطوة لكي يعمل البرنامج مع قاعدة البيانات. اذهب إلى تبويب **Environment** في Render وأضف المفاتيح التالية:

- `SUPABASE_URL`: (احصل عليه من إعدادات Supabase).
- `SUPABASE_ANON_KEY`: (احصل عليه من إعدادات Supabase).
- `SUPABASE_JWT_SECRET`: (موجود في Supabase -> Settings -> API).
- `PORT`: `3005` (أو اتركه وسيتعرف عليه Render تلقائياً).
- `NODE_ENV`: `production`.

## ثالثاً: تحديث قاعدة البيانات (Supabase SQL)

تأكد من تنفيذ ملفات الهجرة (Migrations) الموجودة في مجلد `migrations/` داخل **SQL Editor** في Supabase، خاصة الملف الأخير:

- `migrations/012_add_email_to_clients.sql` (لإضافة عمود البريد الإلكتروني للعملاء).

## رابعاً: الدخول للمنصة

- بمجرد اكتمال الـ Build في Render، سيعطيك رابطاً مثل: `https://malaf-platform.onrender.com`.
- هذا هو الرابط الذي ستفتحه وتشاركه مع المستثمرين.
