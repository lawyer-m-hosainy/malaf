# 📋 خطة عمل يوم السبت (9 / 5)

## 1️⃣ الإعدادات التشغيلية المتبقية لإصلاح تشغيل السيرفر (Render)
السيرفر حالياً متوقف لأنه يحتاج إلى مفاتيح الأمان ليعمل. في بداية يوم السبت قم بالآتي:
- [ ] افتح حسابك في منصة **Render** واذهب إلى إعدادات مشروع `malaf-platform` ثم قسم **Environment**.
- [ ] أضف المتغير `SUPABASE_JWT_SECRET` (انسخ قيمته من لوحة تحكم Supabase: Settings → API → JWT Secret).
- [ ] أضف المتغير `ENCRYPTION_KEY` (اكتب أي مفتاح عشوائي من 32 حرفاً لتشفير الواتساب).
- [ ] أضف المتغير `SUPABASE_SERVICE_ROLE_KEY` (من إعدادات API في Supabase).
*(بمجرد إضافة هذه المتغيرات سيعمل الموقع تلقائياً)*
- [ ] ضبط رابط الـ Webhook الخاص بالواتساب في Meta ليكون: `https://malaf-platform.onrender.com/api/whatsapp/webhook`

## 2️⃣ الربط الضريبي (الفاتورة الإلكترونية - ETA)
- [ ] استكمال تطوير واجهة `ETAInvoicing.tsx` لتدعم إدخال الرقم الضريبي للمكتب (`tax_registration_number`).
- [ ] إنشاء دالة (Function) لتوليد الـ QR Code الخاص بمصلحة الضرائب المصرية (Base64 TLV Format) حسب المواصفات القياسية.
- [ ] إضافة كود التكويد الخاص بكل خدمة (ETA Activity Codes) داخل الفواتير.

## 3️⃣ التكامل التقني المتقدم (e-Litigation)
- [ ] العمل على ملف `ELitigation.tsx` للربط (أو محاكاة الربط) مع بوابة التقاضي الإلكتروني بوزارة العدل.
- [ ] تحديث حالات القضايا لتشمل `مربوط ببوابة التقاضي` وتحديثها تلقائياً.

## 4️⃣ تحسينات الذكاء الاصطناعي (AI Memory)
- [ ] تطوير `aiHandler.js` ليحتفظ بسياق المحادثة (Conversation Memory) للموكلين والمحامين.
- [ ] ربط الذاكرة بجدول `whatsapp_messages` بحيث يقرأ الـ AI آخر 5 رسائل في المحادثة قبل الرد.

## 5️⃣ الاختبار الميداني (UAT)
- [ ] ربط رقم هاتف تجريبي حقيقي بمنصة واتساب (Sandbox).
- [ ] تجربة إرسال أمر المحامي (`جلسة 1045 تأجلت لـ 15/7`) والتأكد من انعكاسها في لوحة التحكم.
- [ ] تجربة المحادثة مع الذكاء الاصطناعي لاختبار استقرار الـ (Fallback من Gemini إلى Groq).
